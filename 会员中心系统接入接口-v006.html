# 概述

# 协议规则

规则 说明
传输方式 为保证交易安全性,外部采用 HTTPS 传输,内部采用 dubbo 传输
提交方式 采用 POST 方法提交
数据格式 提交和返回数据都为 JSON 格式
字符编码 统一采用 UTF-8 字符编码
签名算法 MD5/SHA256
签名要求 请求和接收数据均需要校验签名

# 数据类型说明

序号 类型 含义
1 Text 可以使用字母、数字及中文
2 Number 只能使用数字, 最大长度 10 位
3 Amount 金额,最大长度 12 位,单位为:分, 例:1.23 元=123
4 DateTime 日期时间。格式为:yyyyMMddHHmmss
5 Date 日期。格式为:yyyyMMdd
6 JSON String 表示一个 JSON 字符串
7 JSON Object 表示一个 JSON 对象
8 JSON Array 表示一个 JSON 数组
9 Boolean 布尔逻辑值
10 Dict 枚举型数据

# 传输数据强制说明

序号 符号 含义
1 M 必须填写的域
2 C 某条件成立时必须填写的域
3 C- 某条件成立时删除的域
4 M+ 必须增加的域
5 O 受理方和发卡方自选填写的域
6 -> 转发域
7 R 必须与先前报文中对应域的值相同的域
